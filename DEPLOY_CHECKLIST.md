# Чеклист деплоя на Beget

## ✅ Перед деплоем

- [ ] Получить Supabase URL и Anon Key
- [ ] Обновить `.env.production` с Supabase данными
- [ ] Создать таблицу `contacts` в Supabase
- [ ] Проверить работу Telegram бота

## 🚀 Процесс деплоя

- [ ] Подключиться к Beget по FTP/SSH
- [ ] Очистить папку `public_html/waterdonetsk.ru/`
- [ ] Загрузить файлы из `dist/`
- [ ] Скопировать `.htaccess.example` как `.htaccess`

## ✅ После деплоя

- [ ] Открыть https://waterdonetsk.ru/
- [ ] Проверить консоль браузера (нет ошибок)
- [ ] Заполнить форму доставки воды
- [ ] Проверить сохранение в Supabase
- [ ] Проверить уведомление в Telegram
- [ ] Убедиться, что пользователь видит сообщение об успехе

## 📦 Файлы для загрузки

- `waterdonetsk-deploy.zip` (архив с файлами)
- Или папка `dist/` со всеми файлами
- `.htaccess.example` (переименовать в `.htaccess`)

## ⚠️ Критические моменты

1. **Supabase должен быть настроен** перед деплоем
2. **Telegram бот** уже настроен и работает
3. **.htaccess** обязателен для правильной работы SPA
4. **Очистка папки** - удалить ВСЕ старые файлы

## 🆘 Если проблемы

1. **Ошибки в консоли:** Очистить кэш браузера (Ctrl+F5)
2. **Форма не работает:** Проверить переменные окружения
3. **404 ошибки:** Проверить `.htaccess` файл
4. **Supabase ошибки:** Проверить RLS настройки и таблицу
